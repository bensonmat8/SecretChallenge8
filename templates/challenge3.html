This is because the funtion is asked to do an add operation on two variables and the first variable given is a string and sencond variable is a integer.

Problem 2

This is because the funtion is asked to do an add operation on two variables and the first variable given is an integer the sencond variable given is an string.

Problem 3
TypeError: can't multiply sequence by non-int of type 'str'
This is because the funtion is asked to do a multiply operation on two variables and the two variables given are string.

Problem 4
TypeError: can't multiply sequence by non-int of type 'list'
This is because the funtion is asked to do a multiply operation on two variables and the two variables given are 'list'.

Problem 5
ValueError: Invalid
First theory would be that function perform_calculation is given an arguement 'innoc' which is not defined in the funtion. And the function is designed to raise a ValueError if it sees something that is not designed, or it could be try-excpet where spelunk is throwing an exception.

Problem 6
TypeError: zip argument #2 must support iteration
Funtion comp_calc is programmed to zip() two lists and perform another operation, but one of the variable given is an integer and not a list. So, the zip() funtion is not able to iterate.

Problem 7
TypeError: can only concatenate list (not "int") to list
3rd item of the first variable given to 'comp_calc' function is a list ([3]). So when 'perform_calculation' is trying to add all the items in the list variables given from 'comp_calc', it is trying to add [3] and 6. You cannot add a list to an integer.

Problem 8
TypeError: unsupported operand type(s) for +: 'int' and 'str'
The goal I guess is to add the values of keys 'one' and 'two', but value of 'one' is integer and value of 'two' is string. You cannot add integer to string.

Problem 9
KeyError: 'one'
'calc_dict' is given a dictionary without the key 'one' and the program is asked to call key 'one' from the dictionary.
